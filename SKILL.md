# Enhance Prompt for Stitch

将模糊的 UI 想法转化为结构化的 Stitch 优化提示词。

## 何时使用此技能

当用户想要以下场景时激活：

- 优化发送给 Stitch 的 UI 提示词
- 改进效果不佳的提示词
- 为简单想法添加设计系统一致性
- 将模糊概念组织成可操作的提示词

## 增强管道

按以下步骤增强任何提示词：

### Step 1: 评估输入

检查用户提示词缺少哪些元素：

| 元素 | 检查项 | 如果缺失... |
|------|--------|-------------|
| **平台** | "web", "mobile", "desktop" | 根据上下文添加或询问 |
| **页面类型** | "landing page", "dashboard", "form" | 从描述推断 |
| **结构** | 编号的区块/组件 | 创建逻辑页面结构 |
| **视觉风格** | 形容词、氛围、感觉 | 添加适当的描述词 |
| **颜色** | 具体值或角色 | 添加设计系统或建议 |
| **组件** | UI 特定术语 | 转换为正确的关键词 |

### Step 2: 检查 DESIGN.md

查找当前项目中是否有 `DESIGN.md` 文件：

**如果存在 DESIGN.md：**

1. 读取文件提取设计系统块
2. 包含调色板、排版和组件样式
3. 在输出中格式化为 "DESIGN SYSTEM (REQUIRED)" 部分

**如果不存在 DESIGN.md：**

1. 在增强提示词末尾添加此提示：

```
---
💡 **提示：** 为保持多个页面的一致性设计，请使用 `design-md` skill 创建 DESIGN.md 文件。
```

### Step 3: 应用增强

使用以下技术转换输入：

#### A. 添加 UI/UX 关键词

用具体组件名称替换模糊术语：

| 模糊 | 增强 |
|------|------|
| "顶部菜单" | "带 Logo 和菜单项的导航栏" |
| "按钮" | "主要行动号召按钮" |
| "项目列表" | "卡片网格布局" 或 "带缩略图的垂直列表" |
| "表单" | "带标签输入字段和提交按钮的表单" |
| "图片区域" | "全宽图片的英雄区域" |

#### B. 放大氛围感

添加描述性形容词来设定基调：

| 基础 | 增强 |
|------|------|
| "现代" | "简洁、最小化，留白充足" |
| "专业" | "精致、可信，带微妙阴影" |
| "有趣" | "活力、活泼，圆角和大胆颜色" |
| "深色模式" | "深色主题，深色背景上的高对比度强调色" |

#### C. 构建页面结构

将内容组织成编号区块：

```
**页面结构：**
1. **Header:** 带 Logo 和菜单项的导航
2. **Hero Section:** 标题、副文本和主要 CTA
3. **内容区:** [描述主要内容]
4. **Footer:** 链接、社交图标、版权
```

#### D. 正确格式的颜色

提到颜色时，格式化为：

```
描述性名称 (#十六进制) 用于功能角色
```

示例：

- "深海蓝 (#1a365d) 用于主要按钮和链接"
- "暖米色 (#faf5f0) 用于页面背景"
- "柔和灰 (#6b7280) 用于次要文本"

### Step 4: 格式化输出

按以下顺序组织增强的提示词：

```markdown
[一页目的和氛围的一行描述]

**设计系统（必需）：**
- 平台：[Web/Mobile]，[桌面/移动优先]
- 主题：[浅色/深色]，[风格描述]
- 背景：[颜色描述] (#十六进制)
- 主要强调色：[颜色描述] (#十六进制) 用于 [角色]
- 主文本：[颜色描述] (#十六进制)
- [其他设计标记...]

**页面结构：**
1. **[区块]:** [描述]
2. **[区块]:** [描述]
...
```

## 输出选项

**默认：** 将增强的提示词作为文本返回给用户复制。

**可选文件输出：** 如果用户请求，写入文件：

- `next-prompt.md` — 用于 `stitch-loop` skill
- 用户指定的自定义文件名

## 示例

### 示例 1: 模糊输入 → 结构化提示词

**用户输入：**

> "帮我做一个登录页面"

**增强输出：**

```markdown
一个简洁、可信的登录页面，带居中表单和微妙的品牌标识。

**设计系统（必需）：**
- 平台：Web，桌面优先
- 主题：浅色，最小化，专业
- 背景：纯白 (#ffffff)
- 表面：柔和灰 (#f9fafb) 用于表单卡片
- 主要强调色：深蓝 (#2563eb) 用于提交按钮和链接
- 主文本：近黑 (#111827) 用于标题
- 次要文本：中灰 (#6b7280) 用于标签
- 按钮：微圆角 (8px)，表单上全宽
- 卡片：柔圆角 (12px)，柔和阴影提升

**页面结构：**
1. **Header:** 居中的最小 Logo
2. **登录卡片:** 居中的表单，含邮箱、密码字段，"忘记密码？"链接
3. **提交按钮:** 主蓝色"登录"按钮
4. **Footer:** "没有账户？注册"链接

---
💡 **提示：** 为保持多个页面的一致性设计，请使用 `design-md` skill 创建 DESIGN.md 文件。
```

### 示例 2: 功能请求 → 目标编辑

**用户输入：**

> "添加搜索栏"

**增强输出：**

```markdown
在头部导航添加搜索栏。

**具体更改：**
- 位置：Header，用户头像右侧
- 样式：药丸形输入框，柔和灰背景 (#f3f4f6)
- 图标：左侧输入框内的放大镜图标
- 占位符："搜索..." 浅灰 (#9ca3af)
- 行为：聚焦时展开，带微妙阴影
- 宽度：默认 240px，聚焦时 320px

**上下文：** 这是目标编辑。只做此更改，保留所有现有元素。
```

## 最佳实践提示

1. **尽早具体** — 模糊输入需要更多增强
2. **匹配用户意图** — 如果用户想要简单，不要过度设计
3. **保持结构化** — 编号区块帮助 Stitch 理解层级
4. **包含设计系统** — 多页面项目一致性是关键
5. **一次只改一处** — 不要捆绑不相关的更改
