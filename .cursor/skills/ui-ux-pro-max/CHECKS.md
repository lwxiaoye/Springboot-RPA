# 预交付检查清单
---

## 设计质量检查

### ✅ 视觉规范
- [ ] **色彩系统**
  - [ ] 所有颜色使用 CSS 变量或 Element Plus 主题色
  - [ ] 主色、强调色符合品牌调性
  - [ ] 文本对比度 ≥ 4.5:1（普通文本）
  - [ ] 禁用纯黑(#000)用于背景

- [ ] **字体排版**
  - [ ] 使用系统字体栈 + 1 个 Web Font
  - [ ] 字号层级清晰（H1→H2→H3→Body）
  - [ ] 行高 1.5-1.75
  - [ ] 中英文混排优化

- [ ] **间距系统**
  - [ ] 使用统一的间距单位（4px/8px/16px）
  - [ ] 模块间距一致
  - [ ] 内容不拥挤

### ✅ 交互体验
- [ ] **可交互元素**
  - [ ] 所有可点击元素都有 `cursor: pointer`
  - [ ] 按钮有 hover 状态
  - [ ] hover 动画时长 150-300ms
  - [ ] active 状态明确

- [ ] **反馈机制**
  - [ ] 操作成功/失败有消息提示
  - [ ] 加载状态显示（loading spinner）
  - [ ] 表单验证实时反馈
  - [ ] 禁用状态样式清晰

- [ ] **过渡动画**
  - [ ] 尊重 `prefers-reduced-motion` 设置
  - [ ] 非必要不使用动画
  - [ ] 动画时长合理（200-300ms 为宜）

### ✅ 响应式设计
- [ ] **断点覆盖**
  - [ ] 375px (Mobile)
  - [ ] 768px (Tablet)
  - [ ] 1024px (Laptop)
  - [ ] 1440px (Desktop)

- [ ] **布局适配**
  - [ ] 移动端单列布局
  - [ ] 平板双列
  - [ ] 桌面三-四列
  - [ ] 没有横向滚动条

- [ ] **触控友好**
  - [ ] 按钮尺寸 ≥ 44×44px
  - [ ] 触摸间距充足
  - [ ] 不依赖 hover（移动端）

### ✅ 无障碍访问 (Accessibility)
- [ ] **键盘导航**
  - [ ] Tab 键顺序合理
  - [ ] 焦点状态可见（outline）
  - [ ] 可以键盘提交表单

- [ ] **屏幕阅读器**
  - [ ] 所有图片有 alt 属性
  - [ ] 按钮有 aria-label
  - [ ] 表单有 label 标签
  - [ ] 语义化 HTML 标签

- [ ] **颜色对比度**
  - [ ] 文本对比度通过校验工具检测
  - [ ] 不依赖颜色传达信息（有图标/文字辅助）

## 代码质量检查

### ✅ Vue 组件规范
- [ ] **组件结构**
  - [ ] 使用 `<script setup>` 语法
  - [ ] 合理拆分组合式函数（composables）
  - [ ] 单文件组件不超过 400 行

- [ ] **命名规范**
  - [ ] 组件名：PascalCase (e.g., `UserCard.vue`)
  - [ ] 变量名：camelCase
  - [ ] 常量：UPPER_SNAKE_CASE

- [ ] **Props 设计**
  - [ ] 有明确的类型定义
  - [ ] 提供默认值
  - [ ] 使用 `defineProps` 而非 `props` 选项

### ✅ 性能优化
- [ ] **渲染优化**
  - [ ] 大数据列表使用虚拟滚动（`el-table-v2`）
  - [ ] 图片懒加载（`loading="lazy"`）
  - [ ] 避免内联函数（use `v-on:click="handler"`）

- [ ] **网络优化**
  - [ ] 资源懒加载
  - [ ] 图标使用 SVG sprite 或字体
  - [ ] 压缩图片

### ✅ 安全性
- [ ] **XSS 防护**
  - [ ] 用户输入使用 `v-html` 时进行转义
  - [ ] 避免内联脚本

- [ ] **权限控制**
  - [ ] 敏感操作有权限校验
  - [ ] API 调用有错误处理

## 部署前检查

- [ ] `npm run build` 无报错
- [ ] 资源体积合理（首屏 JS < 500KB）
- [ ] 图片资源已压缩
- [ ] 环境变量配置正确

### ✅ 浏览器兼容性
- [ ] 现代浏览器测试通过（Chrome, Firefox, Safari, Edge）

- [ ] 降级方案：旧版浏览器显示提示

### ✅ 文档
- [ ] README 包含项目说明
- [ ] API 文档完整
- [ ] 组件使用示例
- [ ] 环境配置说明

## 自动化检查脚本

```bash
# 颜色对比度检查
npm run check-a11y

# 性能审计
npm run lighthouse

# 代码规范
npm run lint

# 单元测试
npm run test:unit
```

## 快速自检命令

```javascript
// 1. 检查是否使用了 emoji
grep -r "😀\|👍\|🚀" src/

// 2. 检查硬编码颜色值
grep -r "#[0-9A-Fa-f]\{6\}" src/components/ | grep -v "variables"

// 3. 检查图片缺失 alt
grep -r "<img" src/ | grep -v "alt="

// 4. 检查 console.log
grep -r "console\.log" src/
```

## 修复优先级

1. 🔴 **必须修复** - 安全漏洞、核心功能异常
2. 🟡 **应该修复** - 无障碍问题、性能瓶颈
3. 🟢 **建议优化** - 代码风格、边缘情况
