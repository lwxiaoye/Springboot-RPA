# RPA协作中枢 - 企业级聊天系统设计方案

## 一、系统定位

**RPA协作中枢** 是专为RPA企业级平台（特别是金融行业）设计的深度集成式协作工具，不是通用聊天软件的复刻，而是将沟通与RPA操作深度融合的"操作入口"。

## 二、核心设计理念

### 2.1 与现有模块深度打通
```
┌─────────────────────────────────────────────────────────────────┐
│                        RPA协作中枢                                 │
├──────────┬──────────┬──────────┬──────────┬──────────┬─────────┤
│ 任务调度  │ 机器人管理 │ 执行日志  │ 录屏管理  │ 流程仓库  │  AI助手  │
│    ↕      │    ↕      │    ↕      │    ↕      │    ↕      │    ↕    │
│  任务卡片  │ 机器人卡片 │  日志分享  │  录屏分享  │  流程卡片  │  智能问答 │
│  状态推送  │  状态告警  │  一键分享  │  时间标注  │  一键执行  │  日志分析 │
│  快捷操作  │  远程操作  │  问题追溯  │  关键帧   │  参数填写  │  智能推荐 │
└──────────┴──────────┴──────────┴──────────┴──────────┴─────────┘
```

### 2.2 金融合规优先
- 所有消息永久归档，不可删除
- 操作全记录，可追溯
- 敏感数据自动脱敏
- 细粒度权限控制

## 三、功能模块设计

### 3.1 基础聊天功能

| 功能 | 说明 |
|------|------|
| 单聊 | 基于系统用户体系的一对一沟通 |
| 群聊 | 项目群、部门群、角色群、临时讨论组 |
| 消息类型 | 文本、图片、文件、链接、@提及、引用回复 |
| RPA卡片 | 结构化任务/机器人/流程卡片，支持快捷操作 |

### 3.2 RPA专属核心功能

| 功能 | 描述 |
|------|------|
| 任务状态推送 | 成功/失败/超时自动推送卡片给负责人 |
| 卡片快捷操作 | 失败任务卡片显示"重试"/"分配"按钮 |
| 任务进度订阅 | 实时接收任务执行进度通知 |
| 机器人状态告警 | 上线/离线/异常自动推送 |
| 日志一键分享 | 执行日志/录屏分享到会话 |
| 聊天内执行任务 | /run 流程名 参数... |
| AI智能助手 | 错误日志分析、流程推荐 |

### 3.3 金融合规功能

| 功能 | 描述 |
|------|------|
| 消息永久归档 | 撤回后后台仍保留 |
| 操作全记录 | 所有操作记录审计日志 |
| 敏感数据脱敏 | 卡号、身份证号自动脱敏 |
| 敏感词过滤 | 金融行业敏感词库 |
| 细粒度权限 | 基于RBAC的角色权限 |

## 四、技术架构

### 4.1 实体设计

```
ChatConversation     - 会话表（单聊/群聊）
ChatMessage         - 消息表
ChatMessageAttachment - 消息附件表
ChatParticipant     - 会话参与者表
ChatReadStatus      - 已读状态表
ChatAuditLog        - 审计日志表
```

### 4.2 WebSocket实时通信
- 消息实时推送
- 在线状态同步
- 任务状态实时通知

### 4.3 消息卡片类型
- TASK_CARD - 任务卡片
- ROBOT_CARD - 机器人卡片
- FLOW_CARD - 流程卡片
- LOG_CARD - 日志卡片
- APPROVAL_CARD - 审批卡片
- ALERT_CARD - 告警卡片

## 七、实现优先级

1. **P0 - 基础必备**
   - 单聊/群聊基础功能
   - 文本消息收发
   - 消息卡片渲染

2. **P1 - RPA核心**
   - 任务状态卡片推送
   - 机器人状态卡片
   - 审计日志联动

3. **P2 - 金融合规**
   - 消息归档
   - 敏感数据脱敏
   - 权限控制

4. **P3 - 扩展增强**
   - AI智能助手
   - 消息摘要
   - 快捷指令
