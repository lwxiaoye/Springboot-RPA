# RPA 系统 - 运行说明
23
## 一、项目概述
这是一个完整的 RPA（机器人流程自动化）系统，包含以下功能模块：
- 用户管理（登录、注册）
- RPA 流程管理
- 机器人管理
- 任务分发与执行
- 执行日志跟踪

## 二、技术栈
- **后端框架**: Spring Boot 2.7.18
- **数据库**: MySQL 8.0
- **持久层**: Spring Data JPA
- **前端**: HTML5 + CSS3 + JavaScript
- **构建工具**: Maven

## 三、运行前准备

### 1. 环境要求
- JDK 17+
- Maven 3.6+
- MySQL 8.0+

### 2. 数据库配置

#### 方法一：使用提供的 SQL 脚本初始化
执行项目中的 `src/main/resources/init-db.sql` 脚本：

```sql
-- 在 MySQL 中执行
source D:\apache-maven-3.9.9\rpa\src\main\resources\init-db.sql
```

#### 方法二：手动创建数据库
```sql
CREATE DATABASE rpa_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3. 修改数据库连接配置

编辑 `src/main/resources/application.yml` 文件，修改数据库连接信息：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/rpa_system?useSSL=false&serverTimezone=UTC&characterEncoding=utf8
    username: root        # 修改为你的数据库用户名
    password: your_password  # 修改为你的数据库密码
```

## 四、启动应用

### 方法一：使用 Maven 启动（推荐）
在项目根目录执行：
```bash
mvn spring-boot:run
```

### 方法二：使用 JAR 包启动
```bash
java -jar target/rpa-1.0-SNAPSHOT.jar
```

### 方法三：在 IDE 中运行
右键点击 `Main.java` 文件，选择 "Run" 或 "Debug"

## 七、API 接口说明

### 用户接口
- `POST /api/user/login` - 用户登录
- `POST /api/user/register` - 用户注册
- `GET /api/user/{id}` - 获取用户信息
- `PUT /api/user/{id}` - 更新用户信息

### 流程接口
- `GET /api/process` - 获取流程列表
- `GET /api/process/{id}` - 获取流程详情
- `POST /api/process` - 创建流程
- `PUT /api/process/{id}` - 更新流程
- `DELETE /api/process/{id}` - 删除流程

### 机器人接口
- `GET /api/robot` - 获取机器人列表
- `GET /api/robot/idle` - 获取空闲机器人
- `GET /api/robot/{id}` - 获取机器人详情
- `POST /api/robot` - 创建机器人
- `PUT /api/robot/{id}/status` - 更新机器人状态
- `DELETE /api/robot/{id}` - 删除机器人

### 任务接口
- `GET /api/task` - 获取任务列表
- `GET /api/task/{id}` - 获取任务详情
- `POST /api/task` - 创建任务
- `POST /api/task/{id}/assign` - 分配任务给机器人
- `PUT /api/task/{id}/status` - 更新任务状态
- `DELETE /api/task/{id}` - 删除任务

### 日志接口
- `GET /api/log` - 获取日志列表
- `GET /api/log?taskId={id}` - 按任务获取日志
- `GET /api/log?processId={id}` - 按流程获取日志

## 八、常见问题

### 1. 数据库连接失败
- 检查 MySQL 服务是否启动
- 确认数据库用户名密码正确
- 确认数据库 `rpa_system` 已创建

### 2. 端口被占用
如果 8080 端口被占用，可以修改配置文件：
```yaml
server:
  port: 8080  # 修改为其他端口，如 8888
```

### 3. 前端页面无法访问
- 确认应用已完全启动（查看控制台日志）
- 清除浏览器缓存后重试
