# 流程管理画布设计器使用说明

## 功能概述

流程管理的设计模块已升级为**可视化画布模式**，支持通过拖拽和连线的方式设计流程。

## 主要特性

### 1. 两种节点类型

#### 方框节点（流程节点）
- **用途**：代表需要执行的具体流程步骤
- **特点**：必须绑定机器人才能执行
- **属性**：
  - 节点名称
  - 步骤类型（数据采集、数据解析、数据加工等）
  - 机器人分类
  - 执行机器人

#### 菱形节点（条件节点）
- **用途**：代表流程中的条件判断分支
- **特点**：不需要绑定机器人，用于控制流程走向
- **属性**：
  - 节点名称
  - 条件表达式（例如：data.count > 100）
  - 条件说明

### 2. 画布操作

#### 添加节点
- 点击工具栏的"添加流程节点"按钮添加方框节点
- 点击工具栏的"添加条件节点"按钮添加菱形节点

#### 移动节点
- 鼠标左键按住节点并拖动即可移动位置

#### 连接节点
1. 将鼠标悬停在节点上，会显示4个连接点（上、下、左、右）
2. 点击连接点并按住鼠标
3. 拖动到目标节点的连接点上释放
4. 自动创建带箭头的连接线

#### 删除节点
1. 点击选中节点
2. 在右侧属性面板点击"删除节点"按钮

#### 编辑节点属性
- 点击节点后，右侧会显示属性编辑面板
- 流程节点可以设置名称、类型、机器人分类和执行机器人
- 条件节点可以设置名称、条件表达式和说明

### 3. 工具栏功能

- **自动布局**：一键整理所有节点的位置，按网格排列
- **清空画布**：清除所有节点和连接线

### 4. 数据兼容性

系统支持新旧两种数据格式：
- **新格式**：包含 nodes 和 edges 的画布数据结构
- **旧格式**：原有的步骤列表格式

当打开使用旧格式保存的流程时，系统会自动转换为画布格式。

## 使用示例

### 示例1：简单线性流程
```
[开始] → [数据采集] → [数据解析] → [数据加工] → [结束]
```

操作步骤：
1. 添加5个流程节点
2. 依次用连接线将它们连起来
3. 为每个节点绑定对应的机器人
4. 保存设计

### 示例2：带条件的流程
```
         → [条件: count > 100] → [处理大数据] →
[开始] →                                    → [结束]
         → [条件: count <= 100] → [处理小数据] →
```

操作步骤：
1. 添加开始节点（流程节点）
2. 添加条件节点（菱形）
3. 添加两个处理节点（流程节点）
4. 添加结束节点（流程节点）
5. 从开始节点连接到条件节点
6. 从条件节点分别连接到两个处理节点
7. 从两个处理节点连接到结束节点
8. 为流程节点绑定机器人
9. 设置条件节点的表达式
10. 保存设计

## 注意事项

1. **机器人绑定**：只有流程节点（方框）需要绑定机器人，条件节点（菱形）不需要
2. **执行顺序**：流程执行时会按照连接线的顺序进行，遇到条件节点会根据条件表达式判断走哪个分支
3. **保存前检查**：保存前请确保所有流程节点都已绑定机器人
4. **连接线方向**：建议从上到下或从左到右连接，保持流程清晰

## 技术实现

- 前端使用原生 SVG 实现画布功能
- 节点数据存储在 `nodes` 数组中
- 连接关系存储在 `edges` 数组中
- 数据以 JSON 格式保存到数据库的 steps 字段

## 后续优化方向

1. 支持更多节点类型（并行、循环等）
2. 支持连接线上的标签（标注条件分支）
3. 支持撤销/重做功能
4. 支持导入导出流程模板
5. 增加流程验证功能（检测环路、孤立节点等）
